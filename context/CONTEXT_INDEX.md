# Context Discovery Index

## Quick Start

- **Project:** FastEdge AssemblyScript proxy-wasm SDK
- **Package:** `@gcoredev/proxy-wasm-sdk-as` v1.2.3 on npm
- **Purpose:** Write CDN filter applications in AssemblyScript that compile to WASM for Gcore's FastEdge platform
- **Origin:** G-Core fork of the Kong proxy-wasm AssemblyScript SDK with FastEdge-specific host APIs
- **Language:** AssemblyScript (subset of TypeScript — no closures over mutable state, no `try/catch`, explicit numeric types)
- **SDK entry:** `assembly/index.ts` (public API) + `assembly/proxy.ts` (wasm entry points)
- **FastEdge APIs:** Environment variables, secrets (with rotation), KV store (get/scan/zrange/zscan/bloom filter), dictionary (large values > 64 KB)
- **Build:** `pnpm run asbuild` → `build/release.wasm` + `build/debug.wasm`
- **Test:** No test framework — verify by building examples
- **Examples:** 17 standalone apps in `examples/`, each independently buildable

## Decision Tree: What Should I Read?

### Understanding the System (New to Codebase)
1. Read `PROJECT_OVERVIEW.md` (~120 lines)
2. Skim `architecture/SDK_ARCHITECTURE.md` (two-layer design + context classes)
3. Browse `examples/helloWorld/assembly/index.ts` for the minimal app pattern

### Writing a New CDN App / Example
1. Browse `examples/helloWorld/` for minimal boilerplate
2. Read `development/BUILD_AND_EXAMPLES.md` (example pattern section)
3. Read `architecture/PROXY_WASM_LIFECYCLE.md` for lifecycle hooks

### Adding a FastEdge Host API
1. Read `architecture/SDK_ARCHITECTURE.md` (FastEdge layer section)
2. Read `reference/HOST_FUNCTIONS.md` (FastEdge-specific APIs)
3. Use `assembly/fastedge/kvStore.ts` as a template

### Modifying Core Proxy-Wasm Behavior
1. Read `architecture/SDK_ARCHITECTURE.md` (core layer section)
2. Read `architecture/PROXY_WASM_LIFECYCLE.md` (dispatch mechanism)
3. Read `reference/HOST_FUNCTIONS.md` for the full ABI surface

### Debugging Host Call Issues
1. Read `reference/HOST_FUNCTIONS.md` (result codes + memory conventions)
2. Read `architecture/SDK_ARCHITECTURE.md` (memory management section)

### Working with Headers
1. Read `architecture/SDK_ARCHITECTURE.md` (StreamContext + HeaderStreamManipulator)
2. Browse `examples/headers/assembly/index.ts` for usage patterns

### Working with KV Store / Secrets / Dictionary
1. Read relevant section of `architecture/SDK_ARCHITECTURE.md` (FastEdge layer)
2. Read `reference/HOST_FUNCTIONS.md` (specific host functions)
3. Browse matching example: `examples/kvStore/`, `examples/variablesAndSecrets/`, `examples/largeDictionary/`

### Changing Build Configuration
1. Read `development/BUILD_AND_EXAMPLES.md`
2. Check root `asconfig.json` and `package.json`

### Adding an Example
1. Browse `examples/helloWorld/` for the template
2. Read `development/BUILD_AND_EXAMPLES.md` (adding new examples section)

### Updating Consumer Docs (`docs/`)
1. Read `fastedge-plugin-source/.generation-config.md` — this drives all generation
2. Edit the relevant section in `.generation-config.md`
3. Run `./fastedge-plugin-source/generate-docs.sh` to regenerate
4. Do NOT hand-edit `docs/` files directly

## Search Tips

- **Don't** read `CHANGELOG.md` linearly — grep for keywords as it grows
- **Grep patterns:**
  - `grep -r "@external" assembly/imports.ts` — find host function declarations
  - `grep -r "proxy_" assembly/imports.ts` — find all proxy ABI functions
  - `grep -r "export" assembly/index.ts` — find public API surface
  - `grep -r "class.*extends" assembly/runtime.ts` — find context class hierarchy
  - `grep -r "getEnv\|getSecret\|KvStore\|getDictionary" assembly/fastedge/` — find FastEdge API usage
  - `grep -r "registerRootContext" examples/` — find all example entry points
